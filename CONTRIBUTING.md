# Contributing

Thanks for considering a contribution!

## Quick start

1) Fork and clone.
2) Create a feature branch.
3) Run tests or a small smoke check.
4) Open a PR with a clear description and rationale.

## Development

- Bootstrap: `make bootstrap`
- Validate: `make validate`
- Smoke test: `make smoke-test`

## Style

- Keep workflow templates deterministic and only patch known fields.
- Avoid editing vendored code under `vendor/ComfyUI/`.

## Reporting issues

Please include:
- OS + Python version
- Steps to reproduce
- Relevant logs or error output
